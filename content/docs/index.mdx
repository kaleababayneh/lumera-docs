## What is Cascade?

Cascade is a **permanent decentralized storage module** (`x/cascade`) built natively into the [Lumera Protocol](https://lumera.io), a Cosmos SDK Layer-1 blockchain secured by CometBFT (Tendermint) consensus. It allows users to store files across a decentralized network of Supernodes using a single on-chain transaction.

Unlike traditional storage solutions that charge recurring fees or pin-dependent IPFS deployments, Cascade leverages a **pay-once, store-forever** state model. Data is chunked, encoded with [RaptorQ](https://en.wikipedia.org/wiki/Raptor_code) erasure coding, and deterministically distributed across the Supernode network. This ensures data availability even in the face of significant node churn.

## Audience Guide

Choose the path that best matches your goals:

### Application Developers
You want to integrate permanent, decentralized storage into your frontend app, backend service, or Cosmos smart contract.
- Build with our official SDKs for [TypeScript](/docs/getting-started/javascript-sdk), [Go](/docs/getting-started/go-sdk), and [Rust](/docs/getting-started/rust-sdk).
- Explore the [Cascade Architecture](/docs/concepts/architecture) to understand the upload lifecycle.
- Start building with the [Browser App Guide](/docs/guides/browser-app).

### Node Operators & Validators
You want to run infrastructure to secure the Lumera network and potentially earn storage fees.
- Set up a base [Validator node](/docs/validators/overview).
- Upgrade an existing node to become a [Supernode](/docs/supernodes/overview) to serve and store Cascade data.

### Protocol Researchers
You want to understand the consensus, data availability layers, and cryptography behind Cascade.
- Read how we use [RaptorQ Erasure Coding](/docs/concepts/erasure-coding).
- Learn about Cosmos interoperability via [Interchain Accounts (ICS-27)](/docs/concepts/interchain-accounts).

## Why Cascade?

| Feature | Cascade (`x/cascade`) | IPFS | Arweave |
|---------|---------|------|---------|
| **Persistence Model** | Permanent (enforced by Lumera consensus) | Pin-dependent | Permanent (storage endowment) |
| **Redundancy** | RaptorQ erasure coding across Supernodes | None (depends on pinning services) | Blockweave replication |
| **Self-healing** | High availability, automatic chunk restoration | Best effort | No active repair mechanism |
| **Interoperability** | IBC-compatible natively | No | No |
| **State Execution** | On-chain fee distribution via `MsgFinalizeAction` | None natively | On-chain fee |
| **Data Availability** | Protocol-guaranteed via Supernode queries | Best-effort | Protocol-guaranteed |
