## What is Lumera Protocol?

[Lumera Protocol](https://lumera.io) is an application-specific Layer-1 blockchain built with **Cosmos SDK** and **CometBFT** consensus. It provides permanent decentralized storage, data verification, and AI inference services through a network of specialized nodes called **Supernodes**.

The native token is **LUME** (`ulume`, 6 decimals). Validators secure the network via delegated proof-of-stake, and Supernodes extend validators with compute-intensive services.

## What is Cascade?

Cascade is the **permanent decentralized storage** layer of Lumera Protocol. It lets applications store files on the Supernode network with a single on-chain transaction.

Unlike traditional storage solutions that charge recurring fees, Cascade follows a **pay-once, store-forever** model. Files are chunked, encoded with [RaptorQ](https://en.wikipedia.org/wiki/Raptor_code) erasure coding, and distributed across the Supernode network — guaranteeing data availability even if individual nodes go offline.

## Why Cascade?

| Feature | Cascade | IPFS | Arweave |
|---------|---------|------|---------|
| **Persistence model** | Permanent (enforced by protocol) | Pin-dependent | Permanent (storage endowment) |
| **Redundancy** | RaptorQ erasure coding across Supernodes | None (depends on pinning services) | Blockweave replication |
| **Self-healing** | Yes — automatic chunk restoration if nodes drop | No | No active repair mechanism |
| **Cosmos native** | Yes — IBC-compatible via Interchain Accounts | No | No |
| **Pricing** | One-time on-chain fee | Free (pinning costs extra) | One-time storage fee |
| **Data availability** | Protocol-guaranteed via Supernode consensus | Best-effort | Protocol-guaranteed |

## Protocol Modules

Lumera's on-chain logic is organized into distinct Cosmos SDK modules:

| Module | Purpose |
|--------|---------|
| **Action** (`x/action`) | Manages the lifecycle of Cascade storage and Sense verification actions |
| **Supernode** (`x/supernode`) | Supernode registration, staking, lifecycle management, and metrics compliance |
| **Audit** (`x/audit`) | Epoch-based Supernode monitoring, metrics reporting, and enforcement |
| **Claim** (`x/claim`) | Token migration from the predecessor chain via cryptographic proofs |

## Key Capabilities

- **Permanent storage**: Upload files with a single on-chain transaction. No subscriptions, no renewal fees — files are retrievable indefinitely.
- **Erasure coding**: Files are split into chunks and encoded using RaptorQ fountain codes (RFC 6330). The original file can be reconstructed from any sufficient subset of encoded symbols, even when some Supernodes are unavailable.
- **Multi-SDK support**: Official SDKs for [TypeScript](/docs/getting-started/javascript-sdk), [Go](/docs/getting-started/go-sdk), and [Rust](/docs/getting-started/rust-sdk).
- **Wallet integration**: Browser-based apps can use [Keplr](/docs/getting-started/client-setup#browser-with-keplr-wallet) and [Leap](/docs/getting-started/client-setup#browser-with-leap-wallet) wallets. [DirectSecp256k1HdWallet](/docs/getting-started/client-setup#nodejs-with-mnemonic) enables programmatic signing for server-side applications.
- **Cosmos interoperability**: Any IBC-connected chain can use Cascade via [Interchain Accounts (ICS-27)](/docs/concepts/interchain-accounts).
