## SDK Architecture

The `@lumera-protocol/sdk-js` library natively orchestrates the complexity of bridging the Lumera AppChain with the off-chain Supernode mesh. It maps executions across three strictly isolated layers:

<Diagram name="sdk-architecture" />

### Blockchain Control Plane
This layer wraps [CosmJS](https://github.com/cosmos/cosmjs) to parse the on-chain consensus state and mutate modules:

- **`x/action` Interface**: Simulates fees, scopes payload sizes, and registers `MsgRequestAction` tasks.
- **`x/supernode` Interface**: Verifies active Supernode sets and queries node health states.
- **State Machine RPC**: Signs and broadcasts all native payloads to the Tendermint RPC.

### Cascade Data Plane
This layer directly handles heavy I/O and off-chain routing:

- **LEP-1 Engine**: An internal module that implements the official Cascade block derivation paths. It leverages universal bindings (N-API inside Node.js, optimized WASM within browsers) to perform BLAKE3 hashing, Zstandard payload compression, and RaptorQ layout preparation.
- **`CascadeUploader`**: Natively resolves the 5-step upload lifecycle, including multi-part posting to the `SN-API`.
- **`CascadeDownloader`**: Evaluates Kademlia mesh states, routes stream pipelines to active nodes, and verifies index checksums.
- **`SNApiClient`**: A direct HTTP/REST bridge to the offline DA Gateway endpoints (`tcp/8002`).

### Cryptographic Signer Space
A unified offline router that safely multiplexes keys:

- **Browser Extensions (Keplr/Leap)**: Recommends and inherits global `OfflineSigner` contexts without extracting private keys.
- **`DirectSecp256k1HdWallet`**: Headless environment extraction from raw BIP39 mnemonics.
- **ADR-036 Standard**: Forces all generated signers to support `signArbitrary` primitives for off-chain SN-API authentication.

## Client Surface Area

The `createLumeraClient()` factory returns a typed `LumeraClient` encapsulating these boundaries:

```ts
const client = await createLumeraClient({ preset: "testnet", signer, address, gasPrice: "0.025ulume" });

// Control Plane (Tendermint Consensus)
client.Blockchain.Action     // getParams(), getAction(), getActionFee()
client.Blockchain.Supernode  // getParams(), getSupernode(), listSupernodes()
client.Blockchain.Tx         // signAndBroadcast(), simulate(), getTx()

// Data Plane (Cascade DA Mesh)
client.Cascade.uploader      // uploadFile(), prepareFile(), registerAction()
client.Cascade.downloader    // download(), downloadFile(), downloadPrivate()
```

```ts
import {

  isKeplrAvailable,

```

## Type Strictness

The SDK provides robust compile-time safety. 
- All `SN-API` responses are typed strictly via [OpenAPI schemas](https://github.com/LumeraProtocol/sdk-js/blob/master/src/types/snapi.gen.ts).
- All `x/action` and `x/supernode` transactions are routed via generated Telescope Protobuf definitions.
