## Installation

The SDK relies natively on `tokio` for async orchestration and `cosmrs` for Cosmos definitions.

```toml title="Cargo.toml"
[dependencies]
lumera-sdk-rs = "0.1"
tokio = { version = "1", features = ["full"] }
```

---

## Architectural Abstractions

The Rust SDK isolates the network topology into three composable interfaces:

| Module | Architectural Bound | Execution Scope |
|---|---|---|
| **`chain`** | **Control Plane** | Synchronous Cosmos execution. Escrows block fees via `MsgRequestAction`. Queries `x/supernode` sets via Tendermint JSON-RPC. |
| **`snapi`** | **Data Mesh Gateway** | Asynchronous DA execution. Binds to `tcp/8002` off-chain gateways. Polling execution states and routing `HTTP 1.1 POST` payloads. |
| **`cascade`** | **Data Plane (Macro)** | Unified orchestrator. Hashes files (`blake3`), signs index manifests, and securely sequences `chain` routines directly into `snapi` routines. |

## Client Setup

You can safely define static bindings programmatically, configuring your Cosmos RPC engines and offline signing keys.

```rust
use lumera_sdk_rs::chain::ChainConfig;
use lumera_sdk_rs::{CascadeConfig, CascadeSdk};

let chain_cfg = ChainConfig::new(
    "lumera-testnet-2",
    "https://grpc.testnet.lumera.io",
    "https://rpc.testnet.lumera.io",
    "https://lcd.testnet.lumera.io",
    "0.025ulume",
);
let cfg = CascadeConfig::new(chain_cfg, "https://snapi.testnet.lumera.io");
let sdk = CascadeSdk::new(cfg);
```

### Environment Overrides

Alternatively, configure global fallbacks safely for containerized workflows:

```rust
use lumera_sdk_rs::{CascadeSdk, SdkSettings};

let settings = SdkSettings::from_env();
let sdk = CascadeSdk::new(settings.to_cascade_config());
```

| Variable | Default State | Purpose |
|---|---|---|
| `LUMERA_CHAIN_ID` | `lumera-devnet` | AppChain network identifier. |
| `LUMERA_GRPC` | `http://127.0.0.1:9090` | Consensus state query engine. |
| `LUMERA_RPC` | `http://127.0.0.1:26657` | Block polling engine. |
| `LUMERA_REST` | `http://127.0.0.1:1317` | Cosmos LCD Fallback. |
| `LUMERA_GAS_PRICE` | — | Execution constraint bound (e.g. `0.025ulume`). |
| `SNAPI_BASE` | `http://127.0.0.1:8080` | Local or public Supernode gateway load balancer. |
| `LUMERA_MNEMONIC` | — | Direct signer offline keychain. |
| `LUMERA_CREATOR` | — | Explicit Bech32 mapping address. |

## File Upload Execution

Instead of a monolithic command, the Rust SDK offers granular sequencing.

```rust
use lumera_sdk_rs::{CascadeSdk, RegisterTicketRequest};
use std::path::PathBuf;

// 1. Multiplex the BIP39 Key (SignDirect & ADR-036 Arrays)
let (chain_sk, arb_sk) = derive_signing_keys(&mnemonic)?;

// 2. Control Plane: Broadcast to the consensus engine
let registered = sdk.register_ticket(
    &chain_sk,
    &arb_sk,
    &creator,
    &PathBuf::from("./my-document.pdf"),
    RegisterTicketRequest {
        file_name: "my-document.pdf".to_string(),
        is_public: true,
        expiration_time: expiration.to_string(),
    },
).await?;

// 3. Data Plane Gateway: Execute payload transmission to SN-API
let task = sdk.upload_via_snapi(
    &registered.action_id,
    &registered.auth_signature,
    &PathBuf::from("./my-document.pdf"),
).await?;

// 4. Poll execution telemetry
let status = sdk.snapi.upload_status(&task).await?;
```

## File Download Execution

Retrieve chunks dynamically from the Kademlia Mesh via explicit Action Tokens.

```rust
// 1. Data Plane: Query node resolution
let down_task = sdk.request_download(&registered.action_id, &arb_sk).await?;

// 2. DA Gateway: Stream raw payload via TCP
let bytes = sdk.snapi.download_file(&down_task).await?;
```

## Toolchain Dependencies

- `cosmrs` — Cosmos SDK execution mappings and Protobuf bindings.
- `reqwest` — Off-chain HTTP stream clients executing via `rustls`.
- `blake3` — Parallelized integrity evaluations for the DA Layer.
