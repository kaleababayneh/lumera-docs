## 1. System Dependencies

Before compiling the Lumera binary from source, ensure your OS environment contains the standard C-compilers and build dependencies required by the Cosmos SDK.

```bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y build-essential jq curl git wget lz4 make gcc
```

<Callout type="warn">
**Resource Constraints**
Building `lumerad` from source will stress test your machine. If you are operating on a low-memory VPS (under 16GB RAM), it is highly recommended to mount a swapfile to prevent Out-Of-Memory (OOM) compiler panics.
```bash
sudo fallocate -l 4G /swapfile
sudo chmod 600 /swapfile && sudo mkswap /swapfile && sudo swapon /swapfile
```
</Callout>

## 2. Install Go

Lumera is written in Go. The Cosmos SDK (`v0.50.x`) strictly mandates Go versions `1.22+`.

```bash
wget -q https://golang.org/dl/go1.23.4.linux-amd64.tar.gz -O /tmp/go.tar.gz
sudo rm -rf /usr/local/go
sudo tar -C /usr/local -xzf /tmp/go.tar.gz
rm /tmp/go.tar.gz
```

Export your GOPATH explicitly:

```bash
cat >> ~/.profile <<'EOF'
export GOROOT=/usr/local/go
export GOPATH=$HOME/go
export GO111MODULE=on
export PATH=$PATH:/usr/local/go/bin:$HOME/go/bin
EOF

source ~/.profile
go version
```

## 3. Compile the `lumerad` Binary

### Clone the Network Source
Pull the canonical source code from the ultimate truth repository. Ensure you checkout the correct governance-approved release tag (e.g. `v1.11.1`).

```bash
git clone https://github.com/LumeraProtocol/lumera.git ~/lumera-src
cd ~/lumera-src
git checkout v1.11.1
```

### Build & Install
Compile the executable. The Go compiler will output the binary to `$GOPATH/bin/lumerad`.

```bash
go build -o ~/go/bin/lumerad ./cmd/lumera
lumerad version
```

## 4. Initialize Node Configuration

Use the binary to spin up your initial `$DAEMON_HOME` directory (usually `~/.lumera`). 

```bash
# Initialize genesis and configuration files
lumerad init "YOUR_MONIKER" --chain-id lumera-testnet-2
```

### Download the Genesis File
The `genesis.json` defines the network's initial state and parameters. Overwrite the default initialized stub:

```bash
wget -O ~/.lumera/config/genesis.json \
  https://raw.githubusercontent.com/LumeraProtocol/lumera-networks/master/testnet-2/genesis.json
```

<Callout type="error">
**Immediate Action Required: Key Backup**
Initialization generates `node_key.json` (P2P identity) and `priv_validator_key.json` (Consensus signing key). These keys **are** your validator. Cryptographically back them up to offline storage immediately.
</Callout>

## 5. Tune Node Parameters

Cosmos SDK configurations are split between two primary files.

### Network Peers (`config.toml`)
To prevent network isolation, you must peer with canonical seed nodes.
```bash
sed -i 's/^seeds *=.*/seeds = "faff7c1350468c53121a669ac40e317a4a70c425@seeds.testnet.lumera.io:26656"/' \
  ~/.lumera/config/config.toml
```

### Minimum Gas & State Pruning (`app.toml`)
Set the minimum gas threshold to deter transaction spam. Additionally, enable aggressive state pruning to prevent rapid NVMe consumption.
```bash
# Enforce gas minimums
sed -i 's/^minimum-gas-prices *=.*/minimum-gas-prices = "0.025ulume"/' \
  ~/.lumera/config/app.toml

# Enable custom block pruning
sed -i \
  -e 's|^pruning *=.*|pruning = "custom"|' \
  -e 's|^pruning-keep-recent *=.*|pruning-keep-recent = "100"|' \
  -e 's|^pruning-interval *=.*|pruning-interval = "10"|' \
  ~/.lumera/config/app.toml
```

## 6. Fast-Sync via Snapshot (Optional but Recommended)

Syncing blocks natively from `height=0` can take days. Instead, pull a Polkachu state snapshot.

```bash
# Clear any existing corrupt state
lumerad tendermint unsafe-reset-all --home ~/.lumera --keep-addr-book

# Stream and unpack the snapshot
curl -o - -L https://snapshots.polkachu.com/testnet-snapshots/lumera/lumera_XXXXXXX.tar.lz4 \
  | lz4 -c -d - | tar -x -C $HOME/.lumera
```
*(Ensure you swap `lumera_XXXXXXX.tar.lz4` with the latest file from Polkachu).*

## 7. Systemd Service

Create a `systemd` configuration to run the daemon continuously in the background and recover from crashes.

```bash
sudo tee /etc/systemd/system/lumera.service > /dev/null <<EOF
[Unit]
Description=Lumera Consensus Daemon
After=network-online.target

[Service]
User=$USER
ExecStart=$(which lumerad) start --home $HOME/.lumera
Restart=always
RestartSec=3
LimitNOFILE=65535

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable lumera
sudo systemctl start lumera
```

Monitor your daemon's synchronization:
```bash
# Check raw logs
sudo journalctl -fu lumera

# Evaluate block catch-up status
lumerad status 2>&1 | jq '.sync_info | {latest_block_height, catching_up}'
```

Once `catching_up` evaluates to `false`, your state machine is in consensus.
