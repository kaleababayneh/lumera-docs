## What is a Lumera Validator?

Lumera is a Layer-1 AppChain secured by the CometBFT (Tendermint) consensus engine via delegated Proof-of-Stake (dPoS). General validators are fully responsible for producing blocks, signing state transitions, executing IBC packets, and voting on network governance.

<Callout type="info">
Validators who additionally wish to provide offline data availability for the Cascade storage module must register as **Supernodes**. This guide strictly covers the base CometBFT validator setup. For Supernodes, refer to the [Supernode Expansion Guide](/docs/supernodes/overview).
</Callout>

## System Requirements

Validators must maintain 100% uptime to avoid being jailed. The recommended hardware thresholds are standard for CometBFT infrastructure.

| Specification | Minimum (Testnet) | Recommended (Mainnet) |
|-----------|---------|-------------|
| **Computing** | 4 vCPU (x86_64 architecture) | 8+ Dedicated cores |
| **Memory** | 8 GB RAM (+ 4 GB swap) | 32 GB RAM |
| **Storage** | 80 GB NVMe (Fast I/O) | 500 GB+ NVMe SSD |
| **Network Limits**| 1 Gbps | 5 Gbps+ (Unmetered) |
| **OS Environment** | Ubuntu 22.04 LTS | Ubuntu 22.04/24.04 LTS |

## Configuration Paths

### Official Manual Configuration
The standard, production-ready path for initializing your validator involves building the binary from source, initializing your `config.toml` state, and syncing through Polkachu snapshots. Proceed to the [Node Setup Guide](/docs/validators/node-setup).

> [!TIP]
> **Community Setup Script**
> For testnet purposes or fast iterations, community members maintain automated bash workflows. You can execute this script to rapidly spin up a testnet validator without manual flag configurations:
> ```bash
> wget -O setup.sh https://raw.githubusercontent.com/kaleababayneh/lumera-validator/main/setup-testnet-validator.sh
> chmod +x setup.sh && ./setup.sh
> ```
