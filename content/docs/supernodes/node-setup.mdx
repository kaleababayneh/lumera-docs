## 1. Prepare the Secondary Server

SSH into your **Supernode server** (this must be physically separate from the Validator host).

```bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y curl wget jq ufw lz4 tar
```

---

## 2. Partition Local Firewalls

The Supernode daemon (`sn-manager`) sits on the public internet to facilitate DHT logic and REST gateway ingress. Strict security groups are required.

```bash
sudo ufw allow 22/tcp comment "SSH Interface"
sudo ufw allow 4444/tcp comment "Internal SN gRPC"
sudo ufw allow 4445/tcp comment "Kademlia P2P DHT"
sudo ufw allow 8002/tcp comment "Public REST DA Gateway"
sudo ufw --force enable
sudo ufw status
```

---

## 3. Expose Validator gRPC (`app.toml`)

Before bounding the Supernode, you must configure your validator to accept queries from the Supernode securely. 

**On your Validator Server**, expose the `gRPC` port:
```bash
# Bind gRPC strictly from localhost to all interfaces
sed -i 's|address = "localhost:9090"|address = "0.0.0.0:9090"|' ~/.lumera/config/app.toml

# Lock down access to ONLY your Supernode IP
sudo ufw allow from <SUPERNODE_IP_ADDRESS> to any port 9090 proto tcp comment "SuperNode Tunnel"

sudo systemctl restart lumera
```

**Back on your Supernode Server**, verify connectivity:
```bash
curl -v telnet://<VALIDATOR_IP_ADDRESS>:9090 --max-time 3 2>&1 | head -5
# Expected state: "Connected to <VALIDATOR_IP_ADDRESS> port 9090"
```

---

## 4. Install `sn-manager` Binary

Download the compiled Go daemon from the canonical repository:

```bash
curl -L https://github.com/LumeraProtocol/supernode/releases/latest/download/supernode-linux-amd64.tar.gz | tar -xz

mkdir -p ~/.sn-manager/bin
install -D -m 0755 sn-manager ~/.sn-manager/bin/sn-manager
```

Inject the binary into your PATH:
```bash
echo 'export PATH="$HOME/.sn-manager/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc

sn-manager version
```

---

## 5. Initialize the DA Node

Construct the daemon's local keyring.

<Callout type="info">
The `sn-manager` manages an entirely independent wallet from your `valoper` consensus wallet.
</Callout>

```bash
export SUPERNODE_PASSPHRASE="your_secure_passphrase"

sn-manager init -y \
  --auto-upgrade \
  --keyring-backend file \
  --keyring-passphrase-env SUPERNODE_PASSPHRASE \
  --key-name supernode \
  --supernode-addr 0.0.0.0 \
  --supernode-port 4444 \
  --lumera-grpc <VALIDATOR_IP_ADDRESS>:9090 \
  --chain-id lumera-testnet-2
```

<Callout type="error">
**Immediate Action Required: Save Mnemonic**
The daemon will output a new 24-word generated mnemonic. Cryptographically back this up offline. This controls your Supernode delegations.
</Callout>

Take note of the generated Supernode account address (`SN_ACCOUNT_ADDRESS` — prefixed with `lumera1...`). You will need this to fund the node and delegate.

---

## 6. Systemd Execution

Register the daemon into system-level background processes for automatic recovery.

```bash
sudo tee /etc/systemd/system/sn-manager.service > /dev/null <<EOF
[Unit]
Description=Lumera Supernode DA Manager
After=network-online.target
Wants=network-online.target

[Service]
User=$USER
Environment=HOME=$HOME
Environment=SUPERNODE_PASSPHRASE=your_secure_passphrase
WorkingDirectory=$HOME
ExecStart=$HOME/.sn-manager/bin/sn-manager start
Restart=on-failure
RestartSec=10
LimitNOFILE=65536

[Install]
WantedBy=multi-user.target
EOF

# Mask the service file to prevent passphrase leaks
sudo chmod 600 /etc/systemd/system/sn-manager.service
sudo systemctl daemon-reload
sudo systemctl enable sn-manager
sudo systemctl start sn-manager
```

Monitor daemon stability:
```bash
sn-manager status
```
*Note: Full network health checks will not pass until the registration payload is broadcasted on-chain.*
