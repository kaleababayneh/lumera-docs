## Cryptographic Authentication (ADR-036)

Because the Supernode Data layer executes completely offline relative to the Tendermint consensus engine, all REST payload mutations must be signed by the native `valoper` or `user` key that originated the on-chain action. 

Lumera achieves this natively via [ADR-036 Arbitrary Signatures](https://docs.cosmos.network/main/architecture/adr-036-arbitrary-signature).
1. Sign the specific task state hash (e.g., the `data_hash` for uploads, or the `action_id` for downloads).
2. Inherit the resulting base64 encoding directly into the `signature` request body.

## DA Ingress: Upload Endpoints

### 1. Initialize DA Pipeline

```http
POST /api/v1/actions/cascade
Content-Type: multipart/form-data
```

**Multipart Execution Fields:**

| Field Key | Primitive Type | Execution Description |
|-------|------|-------------|
| `action_id` | `string` | The consensus event identity. |
| `signature` | `string` | The base64 ADR-036 payload validating the target `data_hash`. |
| `file` | `binary` | The uncompressed byte array. |

**Expected Resolution:**

```json
{
  "task_id": "uuid-task-id" // Ephemeral task tracker token
}
```

<Callout type="warn">
If you receive a `404 Action Not Found`, the Supernode has not yet ingested the latest block from the Tendermint layer. The SDK handles these state delays by retrying up to 5 times.
</Callout>

### 2. Poll Mesh Serialization State

```http
GET /api/v1/actions/cascade/tasks/{task_id}
```

```json
{
  "task_id": "uuid",
  "status": "processing",
  "progress": 45,
  "created_at": "2024-01-01T00:00:00Z"
}
```

## DA Egress: Download Endpoints

### 1. Request Pipeline Reconstruction

```http
POST /api/v1/actions/cascade/{action_id}/downloads
Content-Type: application/json
```

**JSON Execution Field:**
```json
{
  "signature": "base64-encoded ADR-036 signature of the action_id string"
}
```

**Expected Resolution:**
```json
{
  "task_id": "uuid-download-task-id" // Ephemeral download pipeline tracker
}
```

### 2. Stream Kademlia Reconstruction Blocks
Once requested, the active SN-API binds to the Kademlia DHT, locating and pulling redundant RaptorQ chunks.

```http
GET /api/v1/downloads/cascade/{task_id}/status
Accept: text/event-stream
```

To prevent HTTP-long-polling UI freezes, the gateway natively streams block completion states implicitly via Server-Sent Events (SSE).

```text
data: {"status": "processing", "progress": 0}
data: {"status": "processing", "progress": 50}
data: {"status": "completed", "progress": 100}
```

### 3. Retrieve Byte Output

```http
GET /api/v1/downloads/cascade/{task_id}/file
```

**Piping Mechanics:**
The resolution returns `Content-Type: application/octet-stream`. Native browser implementations should bind this directly to a memory-safe `ReadableStream`.

## Exception Codes

| Canonical Code | HTTP Value | State Failure Origin |
|---------|----------|----------|
| `NOT_FOUND` | `404` | The DA layer cannot locate the `x/action` ticket via consensus RPC polling. |
| `UNAUTHORIZED` | `401` | The ADR-036 payload rejected execution due to key mismatches or malformed layouts. |
| `SERVICE_UNAVAILABLE` | `503` | Active Set Kademlia routing timeout. |
