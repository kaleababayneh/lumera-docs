## Overview

Cascade storage is not isolated to the Lumera AppChain. By leveraging the Cosmos [Interchain Accounts (ICS-27)](https://ibc.cosmos.network/v8/apps/interchain-accounts/overview/) protocol, any IBC-connected chain can natively issue storage requests.

This enables powerful cross-chain architectural patterns:
- A DeFi protocol on Injective archiving trade receipts immutably.
- An NFT marketplace on Stargaze routing media storage permanently to Cascade.
- A Cosmos Hub DAO indexing and preserving historical governance proposals off-chain.

## Architectural Flow

<Diagram name="interchain-accounts" />

### Cross-Chain State Transitions
1. **Controller Chain**: The remote chain constructs an IBC `MsgSendTx` packet containing an embedded `/lumera.action.v1.MsgRequestAction`.
2. **Relayer Transit**: An IBC relayer ferries the compiled packet to the Lumera AppChain via an open channel.
3. **Host Chain Execution**: The Lumera ICA host module unwraps the payload and executes it.
4. **Action Confirmed**: The event `action_registered` is emitted. The payload's state within `x/action` transitions to `PENDING`. 

<Callout type="warning">
**Fee Funding**: The ICA alias address on the Lumera side must be pre-funded with sufficient `ulume` to cover the escrow storage fee for the `MsgRequestAction`.
</Callout>

### The `app_pubkey` Delegation
Because an Interchain Account is an algorithmic alias without a true private key mapped to the host chain, it natively cannot sign the off-chain ADR-036 authorizations required by the SN-API. 

The protocol bypasses this by accepting an `app_pubkey` field in the `/lumera.action.v1.MsgRequestAction`:
- A local application on the Controller Chain generates its own keypair.
- It injects the resulting public key into the IBC payload.
- It retains the private key to authorize the off-chain REST uploads to the SN-API gateway.

This isolates the **control plane** (IBC smart contracts) from the heavy **data plane** (gigabyte file streaming).

## SDK & CLI Implementations

### Go SDK Abstractions
The official [Lumera Go SDK](https://github.com/LumeraProtocol/sdk-go) natively packages IBC abstractions for integrators:

```go
import "github.com/LumeraProtocol/sdk-go/client"

// Encode a MsgRequestAction payload for an IBC ICA transit
packedMsg := client.PackRequestForICA(msg)

// Parse incoming ICA acknowledgements to detect the Action ID
actionIDs := client.ExtractRequestActionIDsFromAck(ackData)
```

### Reference CLI Tools
To quickly prototype pipelines, we provide the [Lumera ICA Client](https://github.com/LumeraProtocol/lumera-ica-client):

```bash
# Push payload via ICS-27
lumera-ica-client upload --file archive.tar.gz --controller-chain cosmoshub-4

# Query x/action state bridging
lumera-ica-client action status --action-id <id>
```
